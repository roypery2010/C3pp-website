Object-oriented syntax for C3 without pretending to be a new runtime.

C3PP adds classes, constructors, inheritance sugar, and companion headers, then lowers everything back into plain C3 structs and functions.

Core features

Small surface area, clear lowering rules.

Class syntax

Declare `class Name { ... }` blocks with fields and methods in one place.

Single inheritance

Use `class Child extends Parent` and embed the base struct through `self.super`.

Constructor sugar

Define `fn void init(...)` and call `new ClassName(...)` for heap allocation.

Header generation

Emit companion `.c3i` declarations from `.c3pp` sources or lower standalone `.c3i` inputs.

Compiler pipeline

Write concise syntax. Ship regular C3.

01

Author in C3PP

Use classes, methods, `super`, `new`, and optional declaration-only headers.

02

Lower to C3

Classes become structs, methods become namespaced functions, constructors become helpers.

03

Compile natively

The generated output is handed off to `c3c`, keeping the backend compiler in charge.

Generated shape

The syntax sugar stays honest.

class Dog extends Animal {
    int age;

    fn void init(String name, int age) {
        super.init(name);
        self.age = age;
    }

    fn String speak() {
        return "woof";
    }
}
struct Dog
{
    Animal super;
    int age;
}

fn void Dog.init(Dog* self, String name, int age)
{
    self.super.init(name);
    self.age = age;
}

fn Dog* new_Dog(String name, int age)
{
    Dog* self = std::core::mem::new(Dog);
    self.init(name, age);
    return self;
}

Workflow

Useful commands, ready to paste.

Build the compiler

c3c build c3ppc

Compile a C3PP file

.\c3++c.cmd .\examples\animals.c3pp

Emit generated C3 only

.\c3++c.cmd .\examples\animals.c3pp --emit-only

Write a companion header

.\c3++c.cmd .\examples\animals.c3pp --header

Current limits

Deliberately narrow while the lowering model settles.

Not implemented yet

No virtual dispatch, no interfaces, and no access modifiers.

Inheritance model

Inherited fields are reached through `self.super`, not flattened field access.

Parser posture

The parser stays intentionally small and expects reasonably formatted source.
